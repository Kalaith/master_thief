# Archive - Historical Design Documents

This directory contains historical design and planning documents from the development of Master Thief: Heist Planner.

## Important Note

**These documents describe future features and long-term plans that are NOT yet implemented in the current game.**

For information about what's **actually playable right now**, see the main project documentation:
- **[../../LAUNCH_GUIDE.md](../../LAUNCH_GUIDE.md)** - Complete current game documentation
- **[../../README.md](../../README.md)** - Quick start guide

## Archive Contents

### Design Documents
- **gdd.md** - Game Design Document v3.0
  - Original vision for full idle/gacha RPG
  - 20 character designs with personalities and relationships
  - Complete equipment system design
  - Future features: gacha, guilds, territory control, PvP, etc.

### Implementation Planning
- **IMPLEMENTATION_ROADMAP.md** - Complete Development Plan
  - 4-phase roadmap (22-29 weeks)
  - Resource allocation and timelines
  - Success metrics and milestones
  - Revenue projections

- **phase1.md** - Core Foundation & Basic Gameplay
  - Enhanced character system
  - Class specialization
  - Equipment system
  - Automated heist system
  - Combat & resolution

- **phase2.md** - Gacha & Collection System *(Not yet implemented)*
  - Summoning mechanics
  - Crafting system
  - Advanced progression
  - Monetization features

- **phase3.md** - Advanced Operations *(Not yet implemented)*
  - Multi-team heists
  - Territory control
  - Guild system
  - Character relationships

- **phase4.md** - User-Generated Content *(Not yet implemented)*
  - Custom heist creator
  - PvP competitions
  - Creator economy
  - Esports infrastructure

### Implementation Notes
- **CHARACTER_IMPLEMENTATION_SUMMARY.md** - Notes on implementing the 20 specialists
- **STYLE_UPDATE_SUMMARY.md** - Style guide and theming notes

## What's Currently Implemented

The current game (v1.0.0) includes:
- 20 unique characters from the GDD
- Automated mission system
- Character progression and leveling
- Equipment system (basic implementation)
- Team management
- Auto-save system

See **[../../LAUNCH_GUIDE.md](../../LAUNCH_GUIDE.md)** for complete details on current features.

## Future Development

These archived documents serve as a roadmap for future development. Features from Phase 2-4 may be implemented in future versions:
- Gacha/summoning system
- Guild and social features
- Territory control mechanics
- PvP and competitive features
- User-generated content tools
